# 安装配置fiddler

# 网上下载fiddler4，并进行傻瓜式安装

win7下Fiddler证书安装之后，总是无法抓取https的包； 这是情况最复杂的情况，根据步骤来，都是可以解决的——>

在使用Fiddler抓包时，我们有时需要抓https协议的包，这种需要配置一下 开启监控https才可以

首先 找到Tools——>Options

在弹出的菜单中 选择https项  勾选捕捉https

这样配置完OK之后 一般会弹窗提示安装证书，点击安装，然后重启Fiddler即可。

但有时候没有弹窗安装证书或根本就没有提示，这种情况一般在Win7中较多出现，Win7的系统https方面 在.net Framework4.0上有bug

找到fiddler的安装目录  如: F:\test\fiddler，按住shift键，点击鼠标右键，选择‘在此处打开命令窗口’

# 第一种错误：

# 提示creation of the root certificate was not successful 证书安装不成功

然后执行以下命令 makecert.exe -r -ss my -n "CN=DO_NOT_TRUST_FiddlerRoot, O=DO_NOT_TRUST, OU=Created by http://www.fiddler2.com" -sky signature -eku 1.3.6.1.5.5.7.3.1 -h 1 -cy authority -a sha1 -m 120 -b 09/11/2022

注意：这个命令最后的日期 09/11/2022  一定要大于现在的日期 否则创建证书是过期的

# 第二种错误：

# 出现unable to configure windows to trust the fiddler root certificate提示

如果这一步没有成功，做如下操作：Reset All  Certificates;然后再执行第二步

于是重新到网上下载一个Fiddler证书，下载FiddlerCertMaker到本地，双击安装改文件。地址： http://www.enhanceie.com/dl/FiddlerCertMaker.exe (opens new window)，

安装好之后，点击Actions下面的Open Windows Certicate Manager

说明证书安装好了。查看证书是否安装成功的方法；上面所述第一步装好之后也可以用这个方法查询安装是否成功

到这里安装成功，还是不能抓取HTTPS的包

# 第三步：终极解决方案：

最终发现是引擎的问题，打开Certificates generated by CertEnroll engine;  选择MakeCert,然后 ，点击OK,。重启Fiddler，就可以抓HTTPS的包了。
